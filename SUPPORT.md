# Support Policy

## English
JNDC is supported on a community and best-effort basis.

Current support expectations:

* No SLA, no guaranteed response time, and no guaranteed fix timeline
* No guarantee of backward compatibility for every custom deployment
* No promise that every issue, feature request, or security hardening request will be accepted
* No managed deployment, incident response, or production operations support

Appropriate support channels:

* Use GitHub Issues for reproducible bugs, documentation problems, and concrete feature requests
* Use Pull Requests for fixes or improvements with clear scope
* For sensitive security matters, follow [SECURITY.md](./SECURITY.md)

If you plan to use JNDC in a company or production-like environment, assume you need your own operational ownership, security review, and rollback plan.

## 中文
JNDC 仅提供社区型、尽力而为的支持。

当前支持边界如下：

* 不提供 SLA，不承诺响应时效，也不承诺修复时限
* 不保证适配所有自定义部署，也不保证所有场景下的向后兼容
* 不承诺接受每一个 issue、功能需求或安全加固需求
* 不提供托管部署、生产事故响应或代运营支持

建议使用的支持渠道：

* 可复现缺陷、文档问题、明确的功能需求，请使用 GitHub Issues
* 有清晰范围的修复或改进，请使用 Pull Requests
* 涉及敏感安全问题，请遵循 [SECURITY.md](./SECURITY.md)

如果你计划在企业环境或类生产环境中使用 JNDC，请默认由你自己承担运行维护、安全评审和回滚准备工作。
